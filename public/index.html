How It Works

Three steps to mistake-proof AI

1

Feedback

Give 👍 or 👎 on your AI agent's actions. Feedback is stored in a SQLite+FTS5 lesson DB. On 👎, get corrective actions from similar past failures.

2

Rules

Repeated failures auto-promote into prevention rules. Thompson Sampling adapts which rules fire.

3

Gates

Rules become Pre-Action Gates that block your agent before it repeats the same mistake. No more fix-loops.

FAQ

Common questions

Is this real RLHF?
Honest answer: no. This is context engineering plus enforcement, not RLHF weight training. Feedback becomes searchable memory, prevention rules, and gates that block known-bad actions. Your model's weights are never changed. What actually works: recall injects past context, search_lessons exposes corrective actions, and pre-action gates block known-bad tool calls. What does not work: thumbs up/down changing model weights, or agents self-correcting without context injection.
What's the tech stack?
SQLite+FTS5 lesson DB for fast full-text search. MemAlign-inspired dual recall (principle-based rules + episodic context). Thompson Sampling for adaptive gate sensitivity per failure domain. LanceDB + Apache Arrow for local vector search with Hugging Face embeddings. ContextFS for context assembly. Bayesian belief updates on each memory. All local-first — no cloud required.
What AI agents and editors does this work with?
ThumbGate works with Claude Code, Cursor, Codex, Gemini CLI, Amp, OpenCode, and any other MCP-compatible agent. Cursor ships with a plugin bundle in this repo. VS Code works when you run an MCP-compatible agent inside it, but this repo does not ship a standalone VS Code extension today.
Do I need a cloud account?
No. The free tier runs entirely locally — no cloud, no account, no signup. Pro adds optional cloud sync and team sharing, but local-first is always available.
How are gates different from prompt rules?
Prompt rules are suggestions that agents can ignore. Pre-Action Gates are enforcement — they block the action before execution. Gates are auto-generated from your feedback and use Thompson Sampling to adapt over time.
