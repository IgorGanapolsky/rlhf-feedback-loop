  <action>
Create scripts/validate-feedback.js as CommonJS (no shebang dependency). Port all logic from Subway's validate-feedback.js with these rlhf-specific adaptations:

1. **Path resolution**: Use RLHF_FEEDBACK_DIR env var (same pattern as feedback-loop.js):
   ```js
   const DEFAULT_FEEDBACK_DIR = path.join(__dirname, '..', '.claude', 'memory', 'feedback');
   function getFeedbackDir() { return process.env.RLHF_FEEDBACK_DIR || DEFAULT_FEEDBACK_DIR; }
   ```
   Derive FEEDBACK_LOG, VALIDATION_LOG, QUALITY_REPORT from getFeedbackDir() at call time (not module init).

2. **Schema adaptations** — rlhf uses different field names than Subway:
   - REQUIRED_FIELDS: ['timestamp', 'signal', 'id'] (not 'feedback'/'source' — rlhf uses 'signal')
   - VALID_SIGNAL_VALUES: ['positive', 'negative'] (rlhf uses normalized signals)
   - Keep reward validation: reward must be number in [-1, 1]
   - Keep timestamp validation (ISO 8601 + not future)
   - Remove 'source' field validation (rlhf doesn't have source field)

3. **Semantic validation** — adapt for rlhf signal names:
   - signal === 'positive' && reward < 0 → error
   - signal === 'negative' && reward > 0 → error
   - context too short (< 5 chars) → warning
   - placeholder text detection (TODO/FIXME/placeholder/test/example) → warning
   - tool_name validation against ['Read','Write','Edit','Bash','Glob','Grep','Task','WebFetch'] → info

4. **Anomaly detection** — keep all 3 checks exactly as Subway:
   - Rapid burst: >5 entries within 1 minute of current → warning
   - Duplicate: same context+signal+tool_name exists → warning
   - Skew: >95% positive or <5% positive across allEntries (when >=10) → info/warning
   - Sensitive data patterns: /api[_-]?key/i, /password/i, /secret/i, /token/i, /bearer/i, /\b[A-Za-z0-9]{32,}\b/ → error

5. **Self-correction** (generateCorrections): auto-correct reward to match signal ('positive' → 1, 'negative' → -1), add missing timestamp.

6. **validateEntry(entry, allEntries=[])**: returns { valid, entry, issues, corrections, correctedEntry }. valid=true iff no error-level issues.

7. **Module exports**: `module.exports = { validateEntry, validateSchema, validateSemantics, detectAnomalies, generateCorrections, applyCorrections, loadFeedbackLog };`

8. **CLI** (main() only runs when called directly via `require.main === module`):
   - `--audit`: calls auditFeedbackLog(), writes validation-issues.jsonl + quality-report.json to getFeedbackDir()
   - `--stats`: reads quality-report.json from getFeedbackDir(), prints JSON
   - stdin pipe: parse JSON → validateEntry() → output corrected entry or issues

9. auditFeedbackLog() writes quality-report.json with: {total, valid, invalid, corrected, issuesByLevel, issuesByField, validityRate, auditedAt}

Keep the code clean CommonJS — no ES module syntax.
